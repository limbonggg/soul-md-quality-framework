# SOUL.md Quality Validation Rubric - Quick Reference

## Scoring Scale
- 5 = Exceptional (template-worthy)
- 4 = Strong (minor gaps)
- 3 = Adequate (notable weaknesses)
- 2 = Weak (will cause problems)
- 1 = Critical (broken/missing)

## Gate Thresholds
- PASS: >= 4.0 average
- CONDITIONAL: 3.0-3.9
- FAIL: < 3.0

## Dimensions Quick Check

### 1. Identity Clarity
- [ ] First line defines role
- [ ] IS and IS NOT boundaries explicit
- [ ] No competing identities

### 2. Anti-Sycophancy
- [ ] Zero-invention rule with examples
- [ ] Counterfactual reasoning required
- [ ] Calibrated confidence thresholds
- [ ] Sycophancy trap handling

### 3. Actionability
- [ ] Rules use MUST/MUST NOT/WHEN-DO
- [ ] No "should try" or "aim for"
- [ ] Every rule is testable

### 4. Token Efficiency
- [ ] Word count 600-800
- [ ] No redundancy
- [ ] No tutorial creep

### 5. Domain Specificity
- [ ] References specific tools/frameworks
- [ ] Includes decision trees
- [ ] Uses correct terminology

### 6. SCAN Effectiveness
- [ ] Scenarios, not recall questions
- [ ] Includes conflicting signals
- [ ] Expected behavior specified

### 7. Conflict Resolution
- [ ] Priority hierarchy defined
- [ ] Tie-breaker is mechanical
- [ ] Escalation path exists

### 8. Mode Triggers
- [ ] Entry conditions are keyword-based
- [ ] Behavioral changes per mode
- [ ] Exit conditions defined

## Common Pitfalls (P1-P6)
- P1: Vague reward thresholds
- P2: Missing tie-breaker
- P3: Recall-only SCAN
- P4: Conceptual triggers
- P5: Identity/operations mixing
- P6: Local-only research
